# Frontend v0.7.0 — Release Report

## Overview
- Реліз закриває всі задачі з `docs/plan/v7/frontend07.md`: модуль маркетплейсу, Activity Console 2.0, autosave профілю, telemetry service, мобільна адаптація та оновлені UI-токени.
- Стек: Vue 3, Pinia, Vue Router, Tailwind-based дизайн-система (`src/styles/m4sh.css`, `src/ui/tokens.css`), axios.
- Фокус релізу ― публічний discovery-досвід для студентів, прозорий ActivityLog і надійне редагування профілю.

## Marketplace module
- **Структура**: `/src/modules/marketplace/{api,store,views,components}`.
- **MarketplaceListView.vue**:
  1. Повний набір фільтрів (subjects, price, experience, language, sort), debounce-пошук (300 мс) та синхронізація з URL query.
  2. Пагінація через cursor (`marketplaceStore.nextPageParams`), `loadMore` додає результати.
  3. Telemetry події: `marketplace.search`, `marketplace.filter.reset`, `marketplace.pagination.more`, `marketplace.click.cta`.
  4. UI: адаптивна панель фільтрів, grid карток, empty/loader стани.
- **MarketplaceTutorView.vue**: детальний профіль (avatar, headline, description, subjects, price, experience, сертифікати) + CTA “Запитати тьютора” з telemetry (`profile.viewed`, `marketplace.click.cta`).
- **Store/API**:
  - `marketplaceStore.js` нормалізує фільтри, тримає cursor, помилки, стани loading/tutorLoading.
  - `api/marketplace.js` звертається до `/marketplace/tutors/` та `/marketplace/tutors/<id>/`.

## Activity Console 2.0
- **ProfileActivityView.vue**:
  - Фільтри: `action`, `entity_type`, `date_from/date_to`, кнопки Apply/Reset, Refresh.
  - Таблиця для desktop + card-mode для мобільних (`md:hidden` / `md:block`), metadata tree через `ActivityMetadataTree`.
  - Cursor pagination + кнопка “Load more”, показ повідомлень про помилки.
- **API**: `api/activity.js` підтримує `limit`, `before`, `cursor`, `action`, `entity_type`, `date_from`, `date_to`.

## Profile autosave & snapshot indicators
- `ProfileEditView.vue`:
  - `watch` з debounce (800 мс) збирає payload і викликає `profileStore.autosaveDraft`.
  - Керує статусами `autosaveStatus` (“Автозбереження…”, “Чернетку збережено о …”, “Є незбережені зміни”, “Не вдалося зберегти”).
  - Кнопка “Відкотити зміни” відновлює збережений snapshot; `lastAutosaveLabel` форматовано локалізовано.
- `profileStore.js`:
  - Додає стани `lastAutosavedAt`, `hasUnsavedChanges`, метод `autosaveDraft` (через `api/profile.js` → `/me/profile/autosave/`).
  - `saveProfile` скидає прапорці після успіху.

## Telemetry service
- `src/services/telemetry.js`:
  - Тимчасова черга подій + батч-відправка у `/activity/events/` (поріг 5 подій або таймер 200 мс).
  - Retry з поверненням подій у чергу (обмеження 50), DEV-логування в консоль.
  - Використовується у Marketplace (search/filter/pagination/cta/view), готовий до інших модулів.

## Mobile readiness & UI tokens
- Activity Console: фільтри й кнопки займають всю ширину на мобільних, card-mode замість таблиці.
- Marketplace: filter-panel та cards-grid адаптивні, CTA доступні на touch.
- `src/styles/m4sh.css` і `src/ui/tokens.css` поповнені brand-токенами (типографіка, spacing, surfaces, shadows) та підключені через `Card.vue`/глобальні стилі.

## Known gaps / tests
- Юніт-тести Vue не додавалися (див. “за бажанням” у плані), сценарії перевірені інтерактивно.
- Наступні кроки v0.7.x: додаткові telemetry точки (Activity Console, autosave), покриття e2e/юнит-тестами.
