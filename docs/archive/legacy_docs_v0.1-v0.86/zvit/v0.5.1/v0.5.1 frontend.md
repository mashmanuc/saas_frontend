✅ **Фронтенд-звіт v0.5.1 та контекст попередніх релізів**

Цей документ описує, що вже реалізовано у фронтенді станом на v0.5.1, які доробки перейшли з попередніх версій і що плануємо робити далі (v0.6+ до першої повної версії).

---

## 1. Історія прогресу (v0.1 → v0.5.1)

| Версія | Ключові результати |
| --- | --- |
| **v0.1–v0.2** | Базова auth-SPA на Vue 3 + Pinia: сторінки логіну/реєстрації, bootstrap auth-store, інтеграція з DRF JWT. |
| **v0.3** | Первинні дашборди (т’ютор/студент), маршрути `/tutor`, `/student`, базові компоненти (`Card`, `Heading`). |
| **v0.4** | Підключення classroom API: список класів, detail, відображення студентів, перші тести інтеграції фронт+бек. (див. звіт v0.4.0). |
| **v0.5.0** | Рефактор router/menu, Pinia store для classroom, i18n ключі, загальні UI-патерни. |
| **v0.5.1** | Повне виконання плану `frontend05`: Classroom Dashboard, Detail з invite-модалкою, спільні Avatar/Badge/Heading, фікси logout/refresh. |

---

## 2. Що дає v0.5.1 (поточний стан)

### 2.1 Classroom модуль
1. **Dashboard `/dashboard/classrooms`**
   - Новий маршрут у меню тьютора. @frontend/src/router/index.js#53-69
   - Grid із `ClassroomCard`, skeleton-loaders, empty state, кнопка “Створити клас” з API POST `/tutor/classrooms/`. @frontend/src/modules/classrooms/views/DashboardClassroomsView.vue#1-109
2. **Store + API**
   - `api/classrooms.js` з `getTutorClassrooms` та `getClassroomDetails`. @frontend/src/api/classrooms.js#1-10
   - `useClassroomStore`: списки + detail (`currentClassroom`, `currentErrorCode`, 401/500 обробка). @frontend/src/modules/classrooms/store/classroomStore.js
3. **Classroom Detail**
   - Екран використовує Pinia store, показує студентів через `StudentListItem` (Avatar + Badge; статуси active/pending/muted). @frontend/src/modules/classrooms/views/ClassroomDetailView.vue
4. **Invite Student UI**
   - Кнопка “Запросити студента” + `InviteStudentModal`: email-валідатор, fake API delay, toasts, success/close. @frontend/src/modules/classrooms/components/InviteStudentModal.vue
5. **Дизайн-система**
   - Спільні компоненти `Avatar`, `Badge`, `Heading`, оновлений `Card`. @frontend/src/ui
   - Student Dashboard також переведений на `Heading` для консистентності.

### 2.2 Інфраструктурні фікси
1. **Auth refresh-loop**  
   - Інтерсептор axios більше не повторює `/auth/refresh/` на 401, робить `forceLogout()` + toast. @frontend/src/utils/apiClient.js#62-95
   - `TopNav` чекає на `auth.logout()` і завжди редіректить на `/auth/login`. @frontend/src/ui/TopNav.vue#92-118
2. **Logout 400**  
   - В бекенді `REFRESH_COOKIE_PATH` розширено до `/api/auth/`, тож cookie доходить і на `/auth/logout/`. @backend/config/settings.py#166-175
3. **Smoke-тести**  
   - `npm run dev` + ручні сценарії (login → dashboard → invite → logout).  
   - `npm run build` без попереджень, бандл ~257 kB gzip.

## 3. Що реалізовано в попередніх версіях і сьогодні переиспользується

- **Auth / Pinia / i18n (v0.1–v0.3):** повноцінний JWT-flow, збереження токена/користувача, рольові маршрути та guard передостання релізів.  
- **Dashboard ядро (v0.3–v0.4):** каркас `PageShell`, меню за ролями, Tutor/Student dashboards, сторінки lessons.  
- **Backend інтеграції (v0.4):** `/api/tutor/classrooms/`, `/api/tutor/classrooms/:id/`, `/student/my_tutor/` — тепер використовуються новим UI.  
- **UI kit (v0.4 → v0.5):** Card, Heading, notify, loader-store, які ми розширили у v0.5.1.

Це означає, що новий наставник уже має стабільну основу: auth, роутинг, classroom-модуль, базовий UI-kit та інтеграцію з головними API.

## 4. Незавершені задачі v0.5 → перехід до v0.6

1. **Профіль користувача `/api/me/profile/`**  
   - Backend: API вже існує (GET/PATCH, settings). Потрібно завершити документацію, тести та підготувати avatar endpoints. @backend/apps/users/api/views.py#153-235
   - Frontend: потрібно створити модуль профілю (Overview / Edit / Settings) з Avatar upload (див. план `frontend06.md`).
2. **Дизайн-систему поширити на інші модулі**  
   - Tutor dashboard, lessons, запити студентів.
3. **Invite Student реальний бекенд**  
   - Email-контур, валідація дублів, статуси інвайтів (зараз тільки фейкова відправка).

## 5. План до першої повної версії (v0.6 → v1.0)

### 5.1 v0.6 — “Profile & Settings”
- **Frontend:**  
  - `/dashboard/profile`, `/dashboard/profile/edit`, `/dashboard/profile/settings`.  
  - Компоненти `ProfileCard`, `AvatarUpload`, `InfoRow`, multiselect предметів, timezone picker.  
  - Інтеграція з `PATCH /api/me/profile/`, `POST/DELETE /api/me/avatar/`, автозбереження налаштувань (theme, language, privacy).  
- **Backend:**  
  - Аватар upload endpoints, S3/local storage інтеграція.  
  - UserSettings API (language, notifications, dark_mode).  
  - Тести + OpenAPI.

### 5.2 v0.7 — “Invite & Student Relations”
- Реальний Invite Student backend (email service, токени, статуси).  
- UI для списку інвайтів/статусів в Classroom Detail.  
- Tutor ↔ Student workflows (прийняття запитів, нотифікації).

### 5.3 v0.8 — “Lessons & Scheduling”
- Перший варіант планера уроків: список майбутніх/минулого, створення placeholders.  
- Інтеграція з календарем (Google/ICS) — базове відображення.  
- Уніфікація timezone та availability.

### 5.4 v0.9 — “Analytics & Payments (MVP)”
- Статистика длЯ репетитора (кількість уроків, активні студенти).  
- Підготовка до білінгу (placeholder UI, API-контракти).  
- Додаткові accessibility й performance-поліровки.

### 5.5 v1.0 — “Stabilization & Release”
- Повне тестування (E2E + integration).  
- Документація (README, onboarding гайд для наставників).  
- CI/CD для фронту (Cloudflare) та бекенду (Fly).  
- Маркетинговий лендинг + продовження roadmap v1.1.
