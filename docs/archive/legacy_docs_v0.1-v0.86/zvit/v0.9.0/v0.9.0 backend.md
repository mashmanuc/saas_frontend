# v0.9.0 Backend — Lessons & Scheduling прогрес

## Lessons API
- Створено окремий модуль `apps.lessons` із REST-ендпоінтами (`POST /api/lessons/`, `PATCH /api/lessons/{id}/reschedule/`, `POST /api/lessons/{id}/cancel/`, `GET /api/lessons/my/`). Підтримано UTC-збереження, конвертацію у локальний час користувача та ліміт 500 подій із прапором `has_more_events` @apps/lessons/api/views.py#1-119.
- Серіалізатори повертають `utc_start/utc_end`, `user_local_start/user_local_end`, інформацію про тьютора та студента, а також валідують діапазони дат і статуси @apps/lessons/api/serializers.py#1-137.

## Audit Log v2
- Додано застосунок `apps.audit` з моделлю `AuditEvent`, сервісом `log_event` та покриттям юніт-тестами сервісу @apps/audit/models.py#1-37 @apps/audit/services.py#1-55 @apps/audit/tests/test_services.py#1-47.
- Інтегровано логування в ключові події auth/profile/relations/lessons та створено інтеграційні тести, що перевіряють audit-трейс для логіна, логаута, оновлення профілю й тьюторських інвайтів @apps/users/api/views.py#287-1209 @apps/users/tests/test_audit_integration.py#1-86.

## Тестування
- Додано інтеграційні тести для створення, перенесення, скасування та вибірки уроків із edge-case (відсутність relation, reschedule cancelled, 500+ подій із `has_more_events`) @apps/lessons/tests/test_lessons_api.py#1-160.

## Наступні кроки
