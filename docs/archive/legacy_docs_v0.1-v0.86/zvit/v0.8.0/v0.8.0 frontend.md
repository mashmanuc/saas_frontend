# Звіт v0.8.0 — Фронтенд

## 0. Огляд
- Фокус іти згідно `FRONTEND v0.8.0 — TECHNICAL SPECIFICATION`.
- Закриті блоки: Relations UI (студент/тьютор), Autosave Profile.
- Частково/не виконані: Marketplace (4.x), Themes QA (5), Dev Playground (6), документація (7) — перенесено як пропозиції на наступну ітерацію.

## 1. Relations UI (пункти 1.1–2 зі специфікації)
### Student dashboard
- реалізовано всі три стани (без тьютора, invited, active) із відповідними CTA та картками;
- інтегровано кнопки Accept/Decline у invited стані, переходи до чату/уроків у active;
- додано підказковий блок (state hints) і локалізації (uk/en) для заголовків, описів та дій;
- забезпечено fallback стан із кнопками переходу в marketplace + компонент пошуку.

### Tutor dashboard
- впроваджені таби All / Invited / Active з хінтами та підрахунком станів;
- для Invited доступні Accept / Decline / Resend, з індикацією loading/pending;
- додані статусні бейджі, timezone блок і нотатки студента;
- порожні стани (empty-states) мають окремі тексти та опис тактичних дій.

## 2. Autosave Profile (пункт 3)
- новий PATCH `/api/me/profile/autosave/` з відправкою лише dirty полів (util `buildDirtyPayload`);
- UI-статуси: “Зміни не збережено”, “Збереження…”, “Чернетку збережено о …”, “Помилка”, “Rate-limited”;
- debounce 800 мс, блокування autosave на 10 c після HTTP 429 без показу помилки;
- Save кнопка блокується при незбережених змінах або rate-limit, manual save викликає DELETE autosave;
- діалог відновлення чернетки (restore) зберігає часову мітку `autosaved_at`;
- нові тести покривають util-и, rate-limit, dirty→clean, restore.

## 3. Marketplace (пункт 4) — стан та пропозиції
- **Що виконано:** підготовлений аналіз вимог, але імплементація (нова пагінація, offline/error states, skeleton, debounce search, cache) перенесена.
- **Пропозиція для v0.9.0:** 
  1. Ввести cursor-based пагінацію в marketplaceStore із `results`, `cursor`, `hasMore`, `filters`.
  2. Додати централізовані UX-states (offline, error, skeleton) та debounce 300 мс для пошуку.
  3. Організувати memory cache (10 хв) для швидкого повернення до списку.
